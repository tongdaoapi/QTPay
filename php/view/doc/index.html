1. 签名算法

签名算法 (MD5)

1. 所有非空参数（不包含sign）按参数名ASCII升序排序，拼接成 key1=value1&key2=value2... → stringA。

2. 末尾拼接上 secret=商户密钥 得到 stringSignTemp。

3. 对stringSignTemp进行MD5运算，得到32位大写sign值。

例子：strtoupper(md5(amount=xxx&appid=xxx&ip=xxx&orderSn=xxx&productId=xxx&returnUrl=xxx&secret=xxx))

重要：回调通知验签时，需接收所有参数(除sign和空值)参与签名，不可写死字段。

2. 统一下单

POST /api/order/createOrder
POST /api/order/createOrder 发起支付，获取支付链接/表单

请求参数

参数名 必填 类型 描述
appid 是 String 商户 AppID
amount 是 String 金额（元）
productId 是 String 支付产品 ID
orderSn 是 String 商户订单号
notifyUrl 是 String 回调地址
returnUrl 是 String 返回地址
ip 是 String 客户端 IP
sign 是 String 签名

📤 成功响应示例

{ "code": 200, "msg": "success", "time": "1779455353", "data": { "url": "https://www.xxx.com" } }

⚠️ 失败响应示例

{"code": 500, "msg": "商户支付通道已关闭", "data": []}

响应字段说明

字段 类型 描述
code String 成功：200
msg String 描述
time String 时间
data.url String 跳转地址

3. 查询支付订单

POST /api/order/queryOrder
POST /api/order/queryOrder 查询订单状态，支持商户单号/支付单号

请求参数

参数名 必填 描述
appid 是 商户 AppID
orderSn 是 订单号（商户订单号 / 平台订单号）
sign 是 签名

✅ 成功响应

{ "code": 200, "msg": "success", "time": "1779455353", "data": { "amount": 100, "merchant_order_sn": "xxx", "order_sn": "xxx", "product_id": 1, "pay_status": 1, "pay_at": "2026-05-22 21:17:20", "callback_status": 1, "callback_at": "2026-05-22 21:17:20" } }

响应字段说明

字段 类型 描述
code String 成功：200
msg String 描述
time String 时间
data.amount String 金额（元）
data.merchant_order_sn String 商户订单编号
data.order_sn String 平台订单编号
data.product_id String 产品 ID
data.pay_status String 支付状态（0：未支付 1：已支付）
data.pay_at String 支付时间
data.callback_status String 回调状态（0：未回调 1：回调成功 2：回调失败）
data.callback_at String 回调时间

4. 支付结果通知

POST (form/json) 商户notifyUrl
POST 回调接口 (商户提供) 异步通知支付结果，需返回 "success" 字符串

📨 回调参数 (部分核心)

参数名 描述
appid 商户 AppID
amount 金额（元）
merchant_order_sn 商户订单号
order_sn 平台订单号
product_id 产品 ID
pay_status 支付状态（0：未支付 1：已支付）
pay_at 支付时间

⚠️ 验签重要提醒

接收回调时，绝对不能写死参与签名的字段，需要动态获取所有非空参数(除sign外) 按规则生成签名比对。
支付中心可能会增加扩展字段，商户系统需兼容。

🔁 响应要求

成功接收 → 输出纯文本: success (不要带任何HTML或JSON)
失败或其它返回值 → 系统会重试回调
签名安全第一 · 请务必保管好商户密钥key · 支持MD5异步验签
